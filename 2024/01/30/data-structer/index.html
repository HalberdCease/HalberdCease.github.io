P2572 [SCOI2010] 序列操作

码长：4.10KB。

不是分块

老牌线段树题目，非常史山。

有区间赋值 0 和 1、区间取反、区间 1 总数、区间连续 1 最大值，线段树维护以下信息：0、1 总数，0、1 区间连续最大值，0、1 左端起最大值和右端起最大值。

同时需要维护区间赋值和区间取反的 tag。

考虑合并以上信息：

  • 区间 0、1 总数直接加即可；
  • 区间左端连续 0 最大值，如果左儿子区间全部是 0，那么就是左边所有点加上右儿子区间左端连续 0 的个数，否则就是左区间左端连续 0 的个数；
  • 区间右端连续 0 最大值，连续 1 最大值同理；
  • 区间 0、1 最大值就是原来左右两区间的最大值，加上中间合并的最大值（如 0 的最大值，就是左区间的、右区间的和左区间右连续加上右区间左连续的最大值）。

打标记，如果是区间推平，直接覆盖所有标记，然后改信息；

如果是区间取反，取反标记异或 1，交换上面所有信息（0、1 互相交换）。

注意细节，然后就没有了。

P8969 幻梦 | Dream with Dynamic

码长：2.92KB。

好听的名字，好深意的题目背景。

看到 popcount()\text{popcount()}popcount() 想到其值域为 log⁡V\log VlogV，考虑在块内维护。

对于整块的 popcount\text{popcount}popcount，当块内全部都被 popcount\text{popcount}popcount 后，其值只有 log⁡V≤64\log V\le 64logV≤64 种可能，因此可以建立每一个位置对其值的映射，后面进行整块修改的时候，只用修改映射的值，单次复杂度 O(log⁡V)O(\log V)O(logV)。

整块的加法直接打 tag 就好了。

对于散块的操作，因为一次操作最多打出 2 个散块，直接暴力即可，但是在整块中维护的性质会被破坏掉，所以直接把映射拆掉，转化成朴素的权值记录，单次 O(B)O(B)O(B)。下一次再次转化成映射的时候，也只会有 O(B)O(B)O(B) 的复杂度，所以可以直接拆。

注意本题的块长，直接取 n\sqrt{n}n​ 是不行的，如果只有 popcount()\text{popcount()}popcount()，会爆炸，需要取 1145 为佳。

P4137 Rmq Problem / mex

码长：2.34KB （咋越来越水了……

不是分块，但也是一款正宗的根号算法。

询问离线，加上删数的递推是显然的，想到莫队算法，因为本题只有删数转移指针 O(1)O(1)O(1) 好做，所以考虑回滚莫队，然后这题就没有了。

说一下回滚莫队的一些细节：询问按左端点所在块升序为第一关键字，按右端点降序为第二关键字进行排序。对于每一个块的询问，左指针都从块的最左边移向左端点，然后再“回滚”回去，也就是说，我们只是把指针和状态复原，而答案并不是在复原的过程中递推出来的，而是开始记录的。我们先移动右指针从右边往左移动，直到这个询问的右端点，这时记录一下当前区间的答案，然后把左指针往右移动到左端点，这一部分都和普通莫队一样，记录完这次询问的答案后，我们将左端点原路返回撤销标记，答案直接赋值为之前记录的答案，然后开始下一次询问。如果遇到新的块直接重构就好了。

时间复杂度的话，如果块长为 bbb，令 n,mn,mn,m 同阶，左指针最多跳 nbnbnb 次，右指针最多跳 n2b\frac{n^2}{b}bn2​ 次，综合下来也许 b=nb=\sqrt{n}b=n​ 有最优复杂度 O(nn)O(n\sqrt{n})O(nn​) 了。
